Quick start

There are three independent pieces of plumbing — they all have to be aligned:

SignalRGB (effects engine, colour source) │ 1..4 virtual "Desktop Wallpaper – Screen N" devices │ placed on the SignalRGB canvas ▼ SignalRGBBridge.exe (UDP receiver + WebSocket server, in your tray) │ ws://127.0.0.1:17320/?screen=N ▼ Wallpaper host (Lively OR Wallpaper Engine — one HTML wallpaper page per monitor)

What the installer set up for you

  • SignalRGBBridge.exe sits in the system tray and converts the plugin's UDP frames into WebSocket pushes. Configurator + Builder + Help (this page) are served from it on 127.0.0.1:17320.
  • SignalRGB plugin in Documents\WhirlwindFX\Plugins\ announces 1–4 virtual devices. The plugin's Glow Grid Base Size + Aspect Ratio control the colour resolution.
  • Lively bundles: signalrgb-glow-screen-1…4 dropped into Lively's Library — one per monitor.
  • Wallpaper Engine bundle: a single signalrgb-glow/ in …\steamapps\common\wallpaper_engine\projects\myprojects\ — assigned multiple times with a different Screen index.

Open the Configurator

From here, right-click the tray icon → Configurator…. The Configurator is where you pick how many screens, the background image, the glow layout, widgets, presets, ambient effects, and audio glow.

1 monitor · Lively

Why this setup

You have one screen and you want the glow on it. This is the simplest path — no canvas split, no Screen index, just one device → one wallpaper.

SignalRGB canvas Lively ┌─────────────────┐ ┌────────────────┐ │ ┌─────────┐ │ │ Screen 1 │ │ │ Screen1 │ │ ── ws ── │ Glow tile │ │ │ device │ │ │ (assigned to │ │ └─────────┘ │ │ Monitor 1) │ └─────────────────┘ └────────────────┘

How to set it up

  1. Configurator: top-right of the tab bar, set Screens: to 1.
  2. In SignalRGB → Layouts, drag Desktop Wallpaper – Screen 1 onto the canvas at the position you want colours sampled from. Resize to taste.
  3. In Lively → Library, right-click SignalRGB Glow – Screen 1 → Set as wallpaper → pick your monitor.
  4. Back in the Configurator: pick a background image (or one from the Library strip), tweak glow strength + dim if needed.

1 monitor · Wallpaper Engine

Why this setup

Same idea as Lively but with Wallpaper Engine. WE ships a single combined bundle; on one monitor you just leave Screen index at Screen 1.

SignalRGB canvas Wallpaper Engine ┌─────────────────┐ ┌────────────────┐ │ ┌─────────┐ │ │ SignalRGB Glow │ │ │ Screen1 │ │ ── ws ── │ Screen index: │ │ │ device │ │ │ Screen 1 │ │ └─────────┘ │ │ → Monitor 1 │ └─────────────────┘ └────────────────┘

How to set it up

  1. Configurator: set Screens: to 1.
  2. SignalRGB → Layouts: drag Desktop Wallpaper – Screen 1 onto the canvas.
  3. Wallpaper Engine → My Wallpapers: assign SignalRGB Glow to your monitor.
  4. In WE's per-wallpaper properties panel, leave Screen index at Screen 1.
  5. If audio glow / spectrum widget doesn't pulse: make sure Audio Processing is on in WE's general settings (Settings → Performance).

2 monitors · Lively (independent)

Why this setup

Two monitors, each with its own glow that follows a different region of the SignalRGB canvas. Pick this when you want the left monitor's glow to follow keyboard / strip LEDs on the left side of your physical setup, and the right monitor to follow the right side.

SignalRGB canvas Lively ┌─────────────────────────────────┐ ┌────────────┐ ┌────────────┐ │ ┌────────┐ ┌────────┐ │ │ Screen 1 │ │ Screen 2 │ │ │Screen1 │ │Screen2 │ │ ws │ Glow tile │ │ Glow tile │ │ │ device │ │ device │ │ ─→ │ → Monitor │ │ → Monitor │ │ └────────┘ └────────┘ │ │ left │ │ right │ └─────────────────────────────────┘ └────────────┘ └────────────┘

How to set it up

  1. Configurator: Screens: = 2.
  2. SignalRGB → Layouts: place Desktop Wallpaper – Screen 1 on the LEFT half of the canvas, Screen 2 on the RIGHT half. Resize each to cover the area you want sampled.
  3. Lively → Library: assign SignalRGB Glow – Screen 1 to your LEFT monitor, Screen 2 to your RIGHT monitor.
  4. In the Configurator, the per-screen tabs (Screen 1 / Screen 2) let you pick a different background image, glow layout and widgets per monitor.
Heads-up: "Screen 1" is which SignalRGB device feeds it — not which physical monitor it has to live on. Activate Screen 1's tile on whichever monitor you want it on.

2 monitors · Wallpaper Engine (single bundle, Screen index)

Why this setup

Wallpaper Engine ships one bundle (SignalRGB Glow). You assign it to both monitors and set a different Screen index per assignment — the wallpaper page reconnects to the matching ?screen=N route on the bridge. No need for two separate bundles like the pre-0.7.2-beta era.

SignalRGB canvas Wallpaper Engine ┌──────────────────────────┐ ┌────────────────────┐ ┌────────────────────┐ │ ┌────────┐ ┌────────┐ │ │ SignalRGB Glow │ │ SignalRGB Glow │ │ │Screen1 │ │Screen2 │ │ ws │ Screen index: │ │ Screen index: │ │ │ device │ │ device │ │ ─→ │ Screen 1 │ │ Screen 2 │ │ └────────┘ └────────┘ │ │ → Monitor left │ │ → Monitor right │ └──────────────────────────┘ └────────────────────┘ └────────────────────┘

How to set it up

  1. Configurator: Screens: = 2.
  2. SignalRGB → Layouts: place Desktop Wallpaper – Screen 1 on the LEFT half of the canvas, Screen 2 on the RIGHT half.
  3. WE → My Wallpapers: drag SignalRGB Glow onto your LEFT monitor. In the per-wallpaper properties panel, leave Screen index at Screen 1.
  4. Drag the same SignalRGB Glow onto your RIGHT monitor. In that assignment's properties, change Screen index to Screen 2.
One bundle, N monitors — the trick is the Screen index per assignment. Both wallpaper instances open separate WS connections to the bridge with ?screen=0 and ?screen=1; the bridge routes the matching SignalRGB device's colours to each.

2 monitors · Spanned / mirrored (one wallpaper across both)

Why this setup

You want one wide image stretched across both monitors with the glow sampled from a single SignalRGB device. Useful for super-wide compositions (long cityscapes, panoramic art) where the cut-out region spans both monitors as a continuous picture.

SignalRGB canvas Lively (span mode) or WE ┌─────────────────────────┐ ┌────────────────────────────┐ │ ┌──────────────────┐ │ │ ↕ SignalRGB Glow ↕ │ │ │ Screen 1 │ │ ws │ Screen 1 — Spanned across │ │ │ device (wide) │ │ ─→ │ both monitors │ │ └──────────────────┘ │ │ (single wallpaper page) │ └─────────────────────────┘ └────────────────────────────┘

How to set it up

  1. Configurator: Screens: = 1 (we only need one device for the spanned image).
  2. SignalRGB → Layouts: place Desktop Wallpaper – Screen 1 across the whole canvas area you want sampled.
  3. Lively: right-click SignalRGB Glow – Screen 1 → Customise → set Wallpaper arrangement to Span across screens. Assign to both monitors.
  4. Wallpaper Engine: right-click on its tray icon → Display Settings → choose Spanned; pick SignalRGB Glow.
  5. Plugin Aspect Ratio → Custom, type your combined resolution (e.g. 3840×1080 for two FHD side-by-side → Custom Cols 128 × Rows 36 gives a clean 32:9-shaped glow grid).
When to pick span over independent: if the underlying picture is one image (panoramic). If the two monitors should show different things (different cut-outs / themes), use the independent setup above instead.

3–4 monitors

Same flow as 2 monitors, just with more devices and tiles. Configurator Screens: set to 3 or 4 and the corresponding tabs appear.

Canvas layout suggestions

  • 3 in a row — divide the canvas into vertical thirds, one device per third.
  • 2+1 (main pair + side) — Screens 1 / 2 across the bulk of the canvas, Screen 3 in a smaller dedicated region.
  • 2×2 grid — Screens 1 / 2 along the top, Screens 3 / 4 along the bottom. Each device samples a quadrant of the effect.
  • 3+1 (triple + side) — Screens 1 / 2 / 3 cover the main triple-monitor block; Screen 4 lives in a smaller region for the side / portrait / control monitor.

Wallpaper Engine: assign the same bundle N times

Each assignment gets a different Screen index in WE's per-wallpaper properties panel (Screen 1 / 2 / 3 / 4).

Ultrawide / non-16:9 monitors

The SignalRGB plugin's Glow Grid Base Size + Aspect Ratio settings control how the colour samples are shaped. By default Aspect Ratio = Auto: the plugin reads each screen's viewport from the bridge (the wallpaper page reports it on connect) and derives the longer side from your selected base size.

3840 × 1080 monitor + Base 64 + Aspect Auto: cols = round(64 × 3840/1080) = 228 rows = 64 Total LEDs = 228 × 64 = 14592 (uses chunked UDP) 3840 × 1080 monitor + Base 64 + Aspect 16:9 (fallback): cols = round(64 × 16/9) = 114 rows = 64 Total LEDs = 114 × 64 = 7296 (still chunked)

What you set on the plugin side

  • Aspect Ratio = Auto: simplest, the bridge tells the plugin your real viewport once the wallpaper has connected once. If it stays on 16:9 → the wallpaper hasn't pushed its viewport yet (re-set the wallpaper in Lively / WE).
  • Aspect Ratio = 32:9 / 21:9 / 9:16: forces a fixed shape regardless of viewport. Useful as a workaround when Auto stays stuck on 16:9.
  • Aspect Ratio = Custom: type Custom Cols × Custom Rows directly. The other Aspect Ratio settings are ignored.
Heads-up: larger grids = more UDP per frame. 14592 LEDs per frame at 30 fps ≈ 1.3 MB/s — chunked transport handles it fine on localhost, but if you see the SignalRGB plugin tick stuttering, drop the base size to 32 or pick a coarser aspect.

Tips & common pitfalls

Wallpaper appears stuck

  • If the glow doesn't react at all: bridge probably isn't running. Look for the tray icon — relaunch SignalRGBBridge.exe if missing.
  • If the glow froze on one colour: the wallpaper might be paused. Lively → tray Pause wallpapers toggles all; WE → right-click WE tray icon → Stop wallpapers.
  • Configurator status badge (top right) tells you the WS state — if it stays connecting…, port 17320 is blocked / another process is holding it.

Audio glow / spectrum widget doesn't pulse

  • Lively: needs --audio in LivelyInfo.Arguments. Bundled since v0.7.8-beta; if you imported an older zip, re-import the new one.
  • Wallpaper Engine: needs "supportsaudioprocessing": true inside general in project.json. Bundled since v0.7.10-beta. Also: WE's Audio Processing setting (Settings → Performance) has to be on.
  • No actual audio playing? No pulses to render. Play music or a video and the spectrum should react.

Widgets don't appear or can't be moved

  • Configurator → Widgets → check the lock toggle. Locked = widgets are static + click-through; unlocked = draggable on the wallpaper itself.
  • Tray icon → 🔓 Unlock widgets (all screens) toggles every active screen at once.
  • If Lively is in click-through mode, dragging works in the Configurator's layout preview but not on the live wallpaper. Switch Lively interaction on if you want to drag on the wallpaper directly.

Wallpaper Engine Workshop subscriptions

If you subscribed to an older per-screen SignalRGB Glow – Screen 1/2/3 from Workshop (pre-v0.7.2-beta): those are obsolete. Unsubscribe in Steam and use the single combined SignalRGB Glow the installer drops into your local myprojects folder.
